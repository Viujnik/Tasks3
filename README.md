# Tasks & Validators Lab (Laba #2)

## Архитектура проекта
Проект представляет собой упрощённую модель системы обработки задач, построенную на принципах инкапсуляции и защиты инвариантов данных. Логика разделена следующая:
+ **src/validators/fields**: Базовый дескриптор.
+ **src/validators/errors**: Псевдонимы для исключений `LenError` и `StatusError`.
+ **src/validators/rules**: Модальные правила валидации (типы, длина, диапазоны, статусы) и протокол `Rule`.
+ **src/tasks_models.py**: Доменная модель `Task`.
+ **src/sources**: Сами классы источников задач.
+ **src/protocols**: Описание контракта для классов источников.

## Технические шоколадки
Реализованы ключевые требования по управлению доступом к атрибутам:
+ **Data Descriptors**: Кастомный дескриптор `ValidatedField` обеспечивает строгую валидацию при записи данных. Любое нарушение (тип, длина, значение) вызывает исключение (`LenError`, `StatusError` и др.).
+ **Non-data Descriptors (@property)**: Вычисляемые свойства `is_on_time` и `summary` позволяют получать состояние объекта "на лету" без хранения избыточных данных в памяти.
+ **Инкапсуляция**: Публичное API класса `Task` отделено от внутреннего состояния (хранение данных в защищенных атрибутах с префиксом `_`).
+ **Lazy Loading**: Источник задач `ConsoleSource` реализован как генераторы через `yield`, что обеспечивает эффективную работу с памятью(но первоначальная причина - именно поочередный вывод команд для каждой таски).

## Модель Task
Объект задачи включает следующие защищенные атрибуты:
+ `task_id`, `task_type`, `description`, `priority`, `status`, `created_at`, `deadline`, `payload`.
+ **is_on_time**: Логическое свойство, вычисляющее успех выполнения на основе статуса и дедлайна.
+ **summary**: Форматированная строка с иконками состояния (✅/⏳).

## Запуск и тестирование
Для работы с проектом используется менеджер пакетов `uv`.

+ **Запуск симуляции**: 
  ```bash
  uv run main
  ```
+ **Проверка тестов**:
  ```bash
  pytest
  ```
